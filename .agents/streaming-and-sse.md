# Streaming and Server-Sent Events (SSE)

This guide covers how LocalAI handles streaming responses for chat completions, text generation, and other endpoints that support incremental output.

## Overview

LocalAI supports streaming via Server-Sent Events (SSE), following the OpenAI API convention. When a client sends `"stream": true`, the server writes incremental chunks to the response as they are generated by the backend.

## SSE Format

Each chunk is written as:

```
data: {"id":"...","object":"chat.completion.chunk","choices":[{"delta":{"content":"hello"}}]}

data: [DONE]
```

The final `data: [DONE]` signals end of stream.

## Implementation Pattern

### Setting Up the SSE Response

```go
func ChatCompletionHandler(cl *config.BackendConfigLoader, ml *model.ModelLoader, appConfig *config.ApplicationConfig) gin.HandlerFunc {
    return func(c *gin.Context) {
        var req schema.OpenAIRequest
        if err := c.ShouldBindJSON(&req); err != nil {
            c.JSON(http.StatusBadRequest, schema.ErrorResponse{Error: &schema.APIError{Message: err.Error()}})
            return
        }

        if req.Stream {
            // Set SSE headers before writing anything
            c.Header("Content-Type", "text/event-stream")
            c.Header("Cache-Control", "no-cache")
            c.Header("Connection", "keep-alive")
            c.Header("Transfer-Encoding", "chunked")
            c.Header("X-Accel-Buffering", "no") // disable nginx buffering
        }

        // ... rest of handler
    }
}
```

### Writing Chunks

Use `c.Stream` from gin to write SSE events:

```go
chunkCh := make(chan schema.OpenAIResponse)

go func() {
    defer close(chunkCh)
    // backend generates tokens and sends them to chunkCh
    err := backend.GenerateStream(ctx, req, chunkCh)
    if err != nil {
        log.Error().Err(err).Msg("stream generation error")
    }
}()

c.Stream(func(w io.Writer) bool {
    chunk, ok := <-chunkCh
    if !ok {
        // Send the [DONE] sentinel
        fmt.Fprintf(w, "data: [DONE]\n\n")
        return false
    }

    data, err := json.Marshal(chunk)
    if err != nil {
        log.Error().Err(err).Msg("failed to marshal chunk")
        return false
    }

    fmt.Fprintf(w, "data: %s\n\n", data)
    return true
})
```

### Channel-Based Token Streaming

Backends that support streaming accept a token channel:

```go
type TokenCallback func(token string, usage TokenUsage) bool

// Usage in backend call:
var tokenChannel = make(chan schema.OpenAIResponse, 512)

callback := func(token string, usage TokenUsage) bool {
    select {
    case <-ctx.Done():
        return false // client disconnected, stop generation
    default:
    }

    chunk := buildChunk(requestID, token, req.Model)
    tokenChannel <- chunk
    return true
}
```

## Client Disconnect Detection

Always respect context cancellation to avoid wasting GPU resources:

```go
ctx := c.Request.Context()

select {
case <-ctx.Done():
    log.Debug().Str("request_id", requestID).Msg("client disconnected, aborting stream")
    return
default:
    // continue
}
```

Inside the token callback, returning `false` signals the backend to stop generation.

## Buffering Considerations

- Set `X-Accel-Buffering: no` to prevent nginx from buffering SSE responses.
- Use a reasonably sized channel buffer (e.g., 512 tokens) to avoid blocking the backend while the HTTP layer flushes.
- Gin's `c.Stream` calls `Flusher.Flush()` automatically after each write.

## Non-Streaming Fallback

When `stream: false`, collect all tokens and return a single JSON response:

```go
if !req.Stream {
    var sb strings.Builder
    for chunk := range tokenChannel {
        if len(chunk.Choices) > 0 {
            sb.WriteString(chunk.Choices[0].Delta.Content)
        }
    }
    resp := buildFinalResponse(requestID, sb.String(), req.Model, usage)
    c.JSON(http.StatusOK, resp)
    return
}
```

## Testing Streaming Endpoints

```go
func TestChatCompletionStreaming(t *testing.T) {
    // Use httptest with a ResponseRecorder that implements http.Flusher
    w := &flushableRecorder{ResponseRecorder: httptest.NewRecorder()}
    req := httptest.NewRequest(http.MethodPost, "/v1/chat/completions", body)

    router.ServeHTTP(w, req)

    lines := strings.Split(w.Body.String(), "\n")
    var chunks []schema.OpenAIResponse
    for _, line := range lines {
        if strings.HasPrefix(line, "data: ") && line != "data: [DONE]" {
            var chunk schema.OpenAIResponse
            json.Unmarshal([]byte(strings.TrimPrefix(line, "data: ")), &chunk)
            chunks = append(chunks, chunk)
        }
    }
    assert.NotEmpty(t, chunks)
}

type flushableRecorder struct {
    *httptest.ResponseRecorder
}

func (f *flushableRecorder) Flush() {
    f.ResponseRecorder.Flush()
}
```

## Common Pitfalls

- **Forgetting `[DONE]`**: Some clients hang waiting for the sentinel. Always send it.
- **Not flushing**: Without `Flush()`, chunks may be batched and delivered late.
- **Ignoring context cancellation**: Leads to continued GPU usage after client leaves.
- **Race on response headers**: SSE headers must be set before the first `c.Stream` call or any write.
